# 本番公開後の改善バックログ

更新日: 2026-07-14

## 現在の前提

- 初回に超長文を投入するより、短い本筋を置いて対話で追記する使い方を優先する
- 検索、巨大な管理画面、多数の独立ビューは増やさない
- 一つの思考状態を正本にし、必要な見え方だけを重ねる

## 今回実装したもの

### 原文との対応関係

- 初回入力と追加入力を`ThinkingGraph.sources`へ逐語保存
- 各ノードが根拠となる`sourceIds`を保持
- 選択ノードから対応原文を確認可能
- AIブリーフ、Markdown、XMind、JSONへ原文対応を持ち出せる

### 可読性

- ルート直下の枝を軽量クラスタとして色分け
- 任意の枝を折り畳み、非表示件数を表示
- 主ルート／全関係の既存切替を維持

### AI役割分担

- Mapper / Reviser: Claude Sonnet 5が意味、親子、前提、関係を判断
- Validator: TypeScriptが参照整合性、型、ID、locked保護を検証
- 軽量モデル: 意味を変えないラベル短縮や重複候補生成にだけ利用可能。現時点では本筋判断へ使わない
- Sonnet 5が利用できない場合の意味判断fallbackはSonnet 4.6までとし、Flash系モデルへ黙って委譲しない

## 後で改善する候補

### P1: 長文Document-to-Graph Compiler

必要になった時点で、次の段階処理を追加する。

1. 原文分割
2. 論点・主体・出来事・制約の抽出
3. 重複統合
4. 主枝・補助枝・Parking Lotへの分類
5. 型付き関係、反論、盲点、昇格候補の生成
6. 原文参照を保ったグラフ統合

開始条件: 1回の入力が数千字を超える利用が日常化する、または現在の地図で重要論点の欠落が繰り返される。

### P1: 原文参照の精密化

- 現在は「1回の入力」を一つの原文単位として保存
- 長文対応時に段落・文・文字位置まで細分化する
- ノード選択時に原文側の該当箇所を強調する

開始条件: 一つの追加入力から多数ノードが生成され、どの一文が根拠か判別しづらくなる。

### P1: クラスタ表示の強化

- 現在の色分けを、必要なら薄い背景枠へ拡張
- クラスタ単位の折り畳みと現在枝フォーカスを検討
- semantic zoomは、ノード数が増えてから検証する

開始条件: 30ノード前後で、色分けと折り畳みだけでは全体を保持できなくなる。

### P2: 品質評価セット

- 和島がXMindで作成した実例5件を正解例として保存
- 本筋一致、重要論点欠落、関係線、盲点、利用可能になるまでの修正回数を評価
- モデルやプロンプト変更時に同じ入力で回帰確認する

## 入れないもの

- 検索を前提としたナレッジ管理UI
- すべてのマッピング手法を同時表示する万能図
- 軽量モデルによる北極星・親子関係・前提・削除判断
- 描画ライブラリ固有JSONを意味の正本にする設計
